Audiveris Installation Manual

Updated :

Downloading Audiveris software

The latest version is always available on the Audiveris download area.

There are two distribution per release:

  • A binary distribution which provides the needed binary components to run the program. Example: audiveris-3.2a-bin-20090925.zip
  • A full distribution which, besides the binary components, provides source code, JavaDoc, build procedures, training material, etc. that is all the source items to modify and rebuild the program. Example: audiveris-3.2a-full-20090925.zip

Just grab your distribution of choice and expand the archive into some directory. Nota: If you are interested in previous releases, you can browse the "archives" subfolder of the download area.

System Requirements

  • OS: Audiveris is essentially a Java application, and as such should be usable on a wide variety of platforms. It is today being developed under Windows and tested under Mac OS, Solaris and Linux.

    Note however that, since release 3.2a, Audiveris uses Google Tesseract OCR for the translation of textual glyphs. Tesseract was developed in C++, and as such its use is OS-dependent. Please refer to the Tesseract OCR section below.

  • Java: Java 6 is highly recommended since it already contains the JAXB 2 component which is needed for XML persistency. If you cannot use Java 6, you can use Java 5 but you will have to download JAXB 2 from the JAXB web site and install it on your own.

Companion tools

Tesseract OCR

Tesseract OCR is a C++ program and Audiveris is a Java application. For the time being, the connection between Audiveris and Tesseract has been implemented for Windows only. (Help would be appreciated from people knowledgeable about this kind of connection).

For Windows environment, the Tesseract "tessdll.dll" component is used, via the Tesjeract Java companion. The Tesseract part needs two specific actions:

  • The installation of Microsoft VC++ 2008. You can use the express edition which is free. VC++ 2005 won't do the job, since we need msvcp90d.dll.
    If VC++ 2008 is not installed, Audiveris will complain when accessing OCR the first time, with a message such as: "Error while loading tessdll.dll" or "Error while loading tesjeract.dll"
  • The definition of a system environment variable, named TESSDATA_PREFIX, and whose value must be <install-dir>/ocr/, including the final "/" or "\" character. For example, my TESSDATA_PREFIX variable has this value: "u:\soft\audiveris\ocr\", without the quotes.

For non-Windows environments, since the connection is not available yet, there will be a warning the first time the OCR class is accessed, but the program will keep on running, without the OCR features (the user will have to enter the text content manually).

Other components

All the other components need no special installation procedure, since their jar files are already provided in the dist/lib folder of the Audiveris installation directory and directly referenced from the manifest of the Audiveris jar file. See the Implementation manual if you need more detailed information on these components.
