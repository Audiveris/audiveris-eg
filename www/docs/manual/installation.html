Getting Audiveris software

The latest version is always available on the Audiveris download area. If you are interested by previous distributions, you can browse the "archives" subfolder of the download area.

System Requirements

Audiveris is essentially a Java application, and as such should be usable on a wide variety of platforms. It is today being developed under Windows XP, and tested under Solaris, Linux and Mac OS.

Beware that Audiveris uses many features from Java 5.0, so you must have at least this version of Java environment available. And since Audiveris uses JAXB 2 for XML persistency you definitively should use Java 6 which already provides JAXB 2.

Companion tools

This section lists all components that are used by Audiveris for various functions.

Java Advanced Imaging (JAI)

JAI is the only companion that you need to install on your own into your JDK or JRE. Please refer to the download and installation information at the JAI web site.

JAI is a comprehensive framework for image processing, of which Audiveris makes a rather limited use.

JAI has its high-level features coded in Java, and low-level primitives delegated to native libraries. It is implemented as a collection of jar files and DLLs (under Windows). It is better installed directly in your JDK (or JRE) by using the installation procedure found at the JAI web-site.

Audiveris uses the latest JAI version (1.1.3-alpha).

Nota: If not correctly installed, the native libraries cannot be used, and JAI issues the following warning :

Could not load mediaLib accelerator wrapper classes. Continuing in pure Java mode.

Installation
  • The download area contains a pair of archives similar to the following ones (the version numbers may be higher than in this example) :
    • audiveris-3.0-20070707.zip contains the various Audiveris components, source and binary, but without the training material.
    • audiveris-3.0-train-20070707.zip is dedicated to the core and whole bases of training material.

You simply need to expand the first archive into a directory of your choice. The second archive (to be expanded in the same directory) is needed only if you want to be able to re-train the Neural evaluator.

Audiveris Directory Structure

Once expanded, the Audiveris archives create the structure described in this section :

  • audiveris-M.m.jar is the application jar file, with "M" and "m" for release numbers.
  • build.xml is a build file, from which the application can be (re)built and launched via Ant
  • bin/
    • audiveris.bat is the application launcher file
  • lib/
      All the companion components have their jar file located in this folder. See the dedicated section.
  • config/
    • neural-network.xml is a optional custom definition of the glyph neural network evaluator, which is created only when you retrain the network. The default definition is provided in the Audiveris jar resource.
    • logging.properties is an optional file which allows to set the logging level of some units.
  • examples/
    • A set of image files that can be used as input examples
  • train/
    • core/
      • A selected subset of the glyph material, representing the "core" of the training material, to be used as an efficient training input for a first re-training from scratch.
    • sheets/
      • The whole base of glyph material composed by all recognized glyphs stored so far. Using this whole base gives the best final results, but is fairly expensive in terms of computing time. It is meant to be used for incremental training, after the core material has been used.
  • www/ This folder directly mirrors the Audiveris web-site on java.net
    • index.html is the web-site home page
    • docs/
      • api/
        • The whole hierarchy of JavaDoc files
      • manual/ The directory contains several html files, used to handle a frame layout. But the main content is stored in the following files:
        • example.html is a quick start guide built around a small example.
        • installation.html is the installation manual you are currently looking at
        • operation.html describes how to operate the application
        • design.html presents the rationale behind several Audiveris constructions (doc not yet available).
        • implementation.html details a few techniques used throughout Audiveris, such as Lags, GlyphLags and Sticks.
      • wiki/
        • notebook.html is a tiddly wiki to handle various notes on Audiveris development.
