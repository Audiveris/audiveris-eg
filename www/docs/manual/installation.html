System Requirements
  • OS: Audiveris is essentially a Java application, and as such should be usable on a wide variety of platforms. It is today being developed under Windows XP and Mac OS, and tested under Solaris and Linux.
  • Java: Java 6 is highly recommended since it already contains the JAXB 2 component which is needed for XML persistency. If you cannot use Java 6, which is not available under Mac OS X as of this writing, you can use Java 5 but you will have to download JAXB 2 from the JAXB web site and install it on your own.

Downloading Audiveris software

The latest version is always available on the Audiveris download area.

Just grab an archive like audiveris-V.V-YYYYMMDD.zip which contains the executable pieces, source code, documentation and training material. Then simply unzip the archive in the directory of your choice.

Nota: If you are interested by previous distributions, you can browse the "archives" subfolder of the download area.

Companion tools

Java Advanced Imaging (JAI)

JAI is the only companion that you need to install on your own into your JDK or JRE.

JAI has its high-level features coded in Java, and low-level primitives delegated to native libraries. Please refer to the download and installation information at the JAI web site since its installation depends on the target operating system. Audiveris uses the latest JAI version (1.1.3 as of this writing).

Nota: If not correctly installed, JAI cannot use the native libraries and will run much slower. It then issues the following warning:
Could not load mediaLib accelerator wrapper classes. Continuing in pure Java mode.

Other components

All the other components need no special installation procedure, since their jar files are already provided in the /lib folder of the Audiveris installation directory and directly referenced from the Audiveris jar file. See the Implementation manual if you need more detailed information on these components.
