Project Goals

Audiveris is an Optical Music Recognition (OMR) module. Starting from the simple bitmap of a music sheet (for example by scanning a paper), it is able to infer the music informations the sheet contains. It can then provide this information to other tools such as a Midi Sequencer, or a Composition Editor, or any other tool that needs music information as input.

There are already commercial tools in this area, such as MusicScan or SharpEye. Audiveris will be, to my knowledge, the first Java open-source OMR tool.

Audiveris works with printed music sheets only, recognition of hand-written scores is at least one order of magnitude harder.

Current Status

Audiveris is still in development, based on the lessons learnt from a prototype written in Ada several years ago.

The final version should comprise 3 main phases:

  1. Automatic extraction of individual music items, essentially based on their appearance and their structure. This is by far the most complex part of the automatic tasks.
  2. Automatic music recognition, based on the extracted music items, and driven by music grammar.
  3. Manual correction of unavoidable remaining errors. This is a rather different domain which could be started in parallel (see foreseen contributions below).

The development so far has mainly focused on phase #1 (automatic music extraction) which now approaches completion. Development of the other two phases has not started yet.

Foreseen Contributions

Here is a preliminary list of domains and tasks for which contribution would make sense and be really appreciated :

  • Integrate with a Visual Music Editor for manual correction, for printing, for playing, etc... For example Xemo (project as seen on NetBeans) would be a natural partnering tool, however the Xemo site doesn't seem active anymore. So this task remains fully open.
  • Use MusicXML as the primary vehicle for exchange of music information with other tools.
  • Move to NetBeans. I'm referring to the platform, not the IDE. This would move Audiveris stand-alone application to a much better integration environment, taking advantage of view and parameter management. This should also ease the integration with visual editors.
  • Consider addressing hand-written scores . This domain could take benefit of a significant part of Audiveris framework, however this would really be a sub-project on its own. To be discussed if anyone is interested by this daunting task.

Current Implementation

Audiveris today is a pure Java stand-alone application, which represents 200 classes and 50_000 lines of Java code.

Techniques
  • Linear Adjacency Graphs (LAG), a technique to record pixel information as a directed graph structure, composed of pixel contiguous runs organized in sections linked by junctions.
  • Sticks , a technique I developed above LAGs to specifically address the structuring sticks of a music sheet (staff lines, ledgers and legato signs are horizontal sticks, while stems and bar lines are vertical sticks).
  • Neural Network, to allow efficient pattern recognition as well as training based on user input for continuous improvement of the tool efficiency.
Technologies
  • Java Advanced Imaging (JAI) for number crunching in image processing.
  • Swing for user interface, with JGoodies enhancements.
  • JFreeChart for charts.
  • Castor for XML persistency.
  • J2SE 5.0, since generics are used wherever the compiler can help.
  • Ant for project building.
